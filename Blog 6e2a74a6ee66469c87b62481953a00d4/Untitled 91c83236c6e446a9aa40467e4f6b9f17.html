动态规划比较适合用来求解最优问题，比如求最大值、最小值等等。

10级楼梯，每次走1级或2级，有多少种走法？

从这个问题出发用动态规划的思想解题。

踏上第10级楼梯，有两种情况

第一种：从第8级 走2⃣️步

第二种：从第9级 走1⃣️步

那么踏上第9级楼梯，也是两种

第一种：从第7级 走2⃣️步

第二种：从第9级 走1⃣️步

递归求解:

function getStairway(n) {
	if (n < 1) return 0
	if (n === 1) return 1
	if (n === 2) return 2

	return getStairway(n - 1) + getStairway(n - 2)
}

getStairway(10)
// 89

递归的方式时间复杂度呢？

用递归树来分析，高度是n - 1，节点个数是2^(n - 1)，那么复杂度可以看成O(2^n)

下面来优化复杂度

发现递归树中有很多重复计算，可以用缓存，即备忘录算法！

const map = new Map()

function getStairway(n) {
	if (n < 1) return 0
	if (n === 1) return 1
	if (n === 2) return 2

	if (map.has(n)) {
		return map.get(n)
	}

	const v =  getStairway(n - 1) + getStairway(n - 2)
	map.set(n, v)
	return v
}

getStairway(10)
// 89
console.log(map)
// Map(8) {3 => 3, 4 => 5, 5 => 8, 6 => 13, 7 => 21, …}
	// 0: {3 => 3}
	// 1: {4 => 5}
	// 2: {5 => 8}
	// 3: {6 => 13}
	// 4: {7 => 21}
	// 5: {8 => 34}
	// 6: {9 => 55}
	// 7: {10 => 89}

从f(1)到f(n)一共n个输入，Map里存了n-2个结果，时间复杂度和空间复杂度都为O(n)

逆向思维

递归的方式是从n到1，那能不能从1到n？

通过迭代的方式，记录n - 1,n - 2的值，得到f(n)的值。

function getStairway(n) {
	let n1 = 1;
	let n2 = 2;
	
	// 从n = 3，开始迭代
	let i = 3;
	let tmp = 0;

	while(n >= i) {
		tmp = n1 + n2;
		n1 = n2;
		n2 = tmp

		i++;
	}

	return tmp
}

getStairway(10)
// 89
